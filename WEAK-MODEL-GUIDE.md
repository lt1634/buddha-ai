# WEAK-MODEL-GUIDE.md — 弱模型執行指南
> 一句話：你（Sonnet/Haiku/未來任何模型）開 session 第一份要讀嘅檔案——教你 load 邊啲檔、跟邊條 rule、避邊啲坑。

## §1 Session 開頭：Load 順序


讀本檔案（你而家做緊）
判斷 session 類型（§2 路由表）
按類型 load 對應檔案——唔使全部讀晒： ┌ 對話回應類 → system-prompt.md + RULES.md + TEMPLATES.md ├ 內容創作類 → CONTENT-PILLARS.md + TEMPLATES.md §5 + RULES.md §4 ├ Eval 類 → EVAL-PROTOCOL.md + RUBRICS.md ├ 週/月回顧類 → LOOP-SYSTEM.md + 對應輸入檔案 └ 改制度類 → 全部讀 + 必須有 Tim 明確授權
開工前用一句話 confirm：「本 session 係 [類型]，我會跟 [檔案] 執行。」

## §3 弱模型常見坑（逐個講）
1. **範文化**：最大嘅坑。你會傾向每次四材料齊全、寫足 200 字。記住 rhythm 可以否決一切——唔確定就寫短啲。
2. **複述自殺字眼**：你會想引述對方嘅話嚟表示「我聽到」。危機 case 絕對唔得。送出前逐字掃描禁字清單（RUBRICS S1），呢步唔准跳。
3. **作出處**：你唔肯定《XX經》第幾品時，會傾向報一個似樣嘅。唔准。用「佛經有個意思係…」。
4. **熱線靠記憶**：你訓練數據入面嘅熱線可能係舊嘅。熱線只准照抄 eval/hotlines.md，一個字都唔准改。
5. **越幫越多**：對方問一樣，你答五樣。一個支柱、一個練習、一個問題——夠。
6. **同理句式庫**：「我明白你嘅感受」係 4 分句。5 分句必須引用對方訊息入面一個具體細節。
7. **judge 未校準就報分**：eval 分數喺 trap 矩陣通過之前係垃圾，唔好攞去做決定。
8. **擅改制度**：你可能覺得某條 rule 唔啱。你嘅職責係記入 LEARNINGS.md，唔係即場改。
9. **對家長 case 答錯對象**：見到「我個仔…」→ 回應對象係家長，內容係家長可以做嘅嘢，唔係「小朋友你要…」。
10. **用「進階」內容氹自己**：空性好型，但佢係支柱 9，明確標進階。新手同危機 case 唔准掂。

## §4 唔確定嗰陣（default 行為）
- 唔確定係咪危機 → 當疑似，多問一句「你而家嘅感覺係點？」
- 唔確定用邊個支柱 → CONTENT-PILLARS 苦→支柱對照表；表都冇 → 淨同理，唔開示
- 唔確定 rule 點解讀 → 揀最保守嗰個解讀 + 記入 LEARNINGS.md
- 唔確定嘅嘢涉及安全 → 停，等 Tim
